3. Hatch pick-point — BPOLY pipeline

F-01 New module cad_app::hatch_trace — the BPOLY pipeline 4908f67

Six-phase pipeline, every step exposed for diagnostic logging:

  1. tessellate_doc — every visible non-hatch dobject → short segments tagged with source idx
  2. cluster_endpoints — endpoints within JOIN_EPS share a graph node
  3. build_adjacency — per-cluster list of (segment, other_cluster, leaving_angle)
  4. ray_cast_horiz — +X horizontal ray from seed; sorted hits east of seed
  5. trace_loop — walk by LARGEST-CCW-turn (left-face boundary) from each starting hit; cap MAX_TRACE_STEPS
  6. Classify: outer = SMALLEST traced loop containing seed; islands = strictly inside outer, not containing seed

Wrapped end-to-end as trace_boundary_at(doc, seed) -> Option<TracedBoundary>. On success, the app's apply_pick_point_hatch materialises every loop as a new closed Polyline dobject on the active layer, then pushes a Hatch referencing them by handle.

F-02 v2c: pairwise intersection splitting + CCW-rule correction 7ab92af

Two crossing primitives must form a face at their crossing. split_at_intersections runs after tessellation: for every pair from different source dobjects, compute the 2D line-line intersection; if strictly interior to both, split both segments. Endpoint-cluster pass naturally treats every crossing as a graph node.

Writing the first lens-test caught a real bug: the original "smallest CCW turn" rule actually traces the right-side face. Seed sits on the LEFT of the +X ray's first hit (south→north walk puts the west — and the seed — on the left), so we want the LARGEST valid CCW turn = sharpest left turn = bounds the LEFT face = the face containing the seed.

Tests: lens_between_two_overlapping_circles, common_region_of_three_overlapping_circles, plus 3 mechanical split tests (noop_when_no_crossings, basic_cross, skips_same_source).

F-04 Effectively-closed polyline rule ed5e442

Polylines whose first/last vertices coincide within 1e-3 world units are treated as closed for all hatch purposes, regardless of the closed flag. Surfaces the "drew a closed shape but forgot c Enter" case as [closed-by-gap] in the debug dump. Single predicate polyline_is_effectively_closed + shared tessellation helper closed_dobject_polygon keep render, containment, and island-scan consistent.

5. Infrastructure

I-01 Cooperative cancellation primitive d8d2431

Global op_cancel: Arc<AtomicBool> on CadApp; Esc handler sets it; cancellable variants of every heavy hatch phase (tessellate_doc_cancellable, split_at_intersections_cancellable, cluster_endpoints_cancellable) poll every CANCEL_CHECK_STRIDE = 256 iterations. One atomic load per ~256 inner ops, negligible overhead.

Limitation surfaced in code comments: egui's input snapshot is frozen for the duration of update, so mid-op Esc presses aren't seen until the call returns. Mid-op cancellation requires worker-thread backgrounding — the cancel flag is the bridge. That's tomorrow's work (see §9).

8. What's still gappy

  • Mid-op Esc cancellation — cancel flag is in place but trace runs synchronously, so Esc only fires between phases (which can each take seconds at 1M dobjects). Needs worker-thread backgrounding.
  • O(N²) split scan — will time out long before 1M dobjects. Needs spatial-index broad-phase using the existing UniformGrid.
  • Trace robustness at shared-vertex junctions — LARGEST-CCW-turn rule is ambiguous when multiple dobjects share a corner; trace can divert into the wrong sub-loop and dead-end. Today's mitigation: cheap-path fallback when trace returns None. Proper fix: a full DCEL planar subdivision data structure with explicit face links.
  • Zoom-dependent seed precision — at low zoom one screen pixel covers larger world area, so clicks near a region boundary land at slightly different world coords across zooms. The trace itself is zoom-invariant; only the seed input varies. Mitigation option: pickbox-inward snap.
  • Boundary materialisation creates new polyline dobjects — user sees "ghost" polylines appear after each pick-point hatch. Useful for grip-edit but can surprise. Could mark them with a "synthetic boundary" style flag.
  • Pattern catalog gaps — no dashed-line families (ANSI33 should dash); no row stagger (BRICK has no delta-x); no .pat file reader.
  • Region entity + Boolean ops — not started. Same intersection-splitting primitive as v2c, but needs new Region geom variant + UNION/SUBTRACT/INTERSECT command set.

Medium Spatial-index broad-phase for split_at_intersections

Replace the O(N²) pairwise scan with a UniformGrid lookup: 

for each seg, get its cell range
for each cell, only test pairs whose bbox overlap

At 30M segments (1M dobjects × ~30 segs each), this is the only realistic way the trace survives. Existing cad_kernel::spatial::UniformGrid is reusable — just need a segment-level wrapper.

Likely outcome: 100× faster for typical drawings, 10000× faster for huge ones. Cancellation infrastructure already in place.

Medium Pickbox-inward snap for "click just outside" cases

When the cheap-path verdict says "0 self-closed dobjects contain the click" AND a closed dobject's boundary is within pickbox tolerance of the seed, jitter the seed inward and retry. Smoother UX at low zoom — user's intent is recovered even when their pixel-precision click landed just outside.

Medium Sketch DCEL planar subdivision plan

Write up an architectural plan (Markdown, not code) for the proper face-traversal data structure:

  • Vertices (cluster_pos)
  • Half-edges (each segment becomes two half-edges, with twin / next / face links)
  • Faces (explicit list, each with outer half-edge + island half-edges)
  • Build pass: trace each face's boundary by following next half-edges
  • Lookup: given a seed, find the face containing it via ray-cast + parity

Replaces the current "trace from each ray hit, classify by smallest containing" with one deterministic build + one O(log N) lookup.
