Internal Storage Audit

A byte-level walkthrough of how RUST_CAD holds a drawing in memory, how it serialises to the RSM binary format, and how the DXF reader fills the same structures from group-coded text. Written for design engineers reviewing the data model before extending it.
Pure Rust workspace 5 crates (kernel · io · snap · app · cli) 138 tests 9 M dobject memory budget audited

§ 1TL;DR — design summary

Three rules govern every piece of state in the data model:

Rule 1 — One source of truth per kind
Every Dobject points to tables on the Document (layers, linetypes, pens, truecolors). The Dobject never duplicates what a table can store once and share.
Rule 2 — Indirect what's heavy, inline what's light
ACI is one byte and goes inline on every Dobject. 24-bit truecolor is a u16 index into a shared, deduplicated table — so a million dobjects all in #FF8040 cost one row in the table, not a million u32s.
Rule 3 — On-disk format is decoupled from in-memory
RSM still inlines truecolor RGB on disk (lossless DXF round-trip). The reader interns each RGB into the loading Document's table; the writer dereferences refs back to inline RGB. Storage refactors don't break files.

What this audit is for

Use this document when:

  • Adding a per-Dobject property — §12 has the seven-step checklist.
  • Estimating memory at a target dobject count — §3 and §6 have the cost tables.
  • Implementing a new DXF record type — §10 has the existing reader's group-code mapping pattern.
  • Modifying RSM — §9 documents the byte-by-byte layout.
  • Reviewing the color inheritance chain — §7 has the resolution diagram.

§ 6Color model

File: cad_kernel/src/color.rs. Refactored 2026-06-02 to separate the per-Dobject color slot from the shared 24-bit RGB pool. 

pub enum Color {
    ByLayer,                  // 0-byte payload — inherits from style.layer
    ByBlock,                  // 0-byte payload — inherits from block (not yet wired)
    Aci(u8),                  // 1-byte ACI palette index (1..=255)
    TrueColorRef(u16),        // 2-byte index into Document::truecolors
}

pub struct TrueColorTable {
    rgbs:   Vec<u32>,                       // index → 0x00RRGGBB
    by_rgb: HashMap<u32, u16>,              // dedup on intern()
}
tag1 B
payload≤ 2 B
pad1 B

Color enum size: 4 bytes (was 8 before the refactor, because TrueColor(u32) forced u32 alignment).

Why indirection

Memory math
Before refactor: per-Dobject color slot = 8 B → 9 M × 8 = 72 MB.
After refactor: per-Dobject color slot = 4 B → 9 M × 4 = 36 MB, plus ~32 B per unique RGB in the dedup table. Typical CAD usage has dozens of unique truecolors; the table overhead is a few KB.

Dedup behaviour

A million Dobjects all in #FF8040 cost:

  • ~4 MB across their per-Dobject color slots (the u16 ref + tag).
  • 4 bytes once in truecolors.rgbs for the RGB.
  • ~32 bytes once in truecolors.by_rgb for the HashMap entry.

API for new code

// Set a per-Dobject TrueColor:
let idx = doc.truecolors.intern(0xFF8040);
dobject.style.color = Color::TrueColorRef(idx);

// Resolve to concrete RGB at render time:
let (r, g, b) = resolve_color(
    dobject.style.color,
    dobject.style.layer,
    &doc.layers,
    &doc.truecolors,
);

§ 7Color inheritance chain

"By Dobject", "By Block", "By Group", and "By Layer" are not separate storage — they are four resolutions of the same Color field, dispatched at render time.

Resolution flow

What each source means

SourceHow it's storedWhen resolved
By Dobject style.color = Color::Aci(n) or Color::TrueColorRef(idx) on the dobject itself. Overrides everything. Render time.
By Block style.color = Color::ByBlock. Resolves to the block's color when the Dobject is inside a block reference. Block support not yet implemented — today ByBlock falls back to ByLayer. Render time, via block→layer chain (when blocks land).
By Group Not a separate color source in the data model. Groups (when added) inherit from the dobjects they contain; group selection is a UX concept, not a color source. —
By Layer style.color = Color::ByLayer. Resolves via layers[style.layer].color. The layer holds the concrete Color::Aci(n) or Color::TrueColorRef(idx). This is the default for new Dobjects. Render time.

Default chain for a freshly drawn dobject

Style::default()
    .color  = Color::ByLayer
    .layer  = LayerTable::LAYER_BASE       // id 0, name "LAYER B"

LayerTable::LAYER_BASE.color = Color::Aci(7)
aci_palette(7)                       = (255, 255, 255)   // white

Hence the project convention: "Dobject by default is drawn on LAYER B (Base) with color 0 (white)."

with_defaults() populates one Layer at id 0 named "LAYER B" (Base). It is reserved — DXF / RSM round-trips would break without it.

Visibility / selectability gates
  • Document::is_visible(idx) = Dobject's own visible AND its layer renders (not hidden, not frozen).
  • Document::is_selectable(idx) additionally requires the layer is not locked.

§ 9.5 — Color encoding

Indirection-agnostic on disk
On-disk format still inlines TrueColor as u32. The writer dereferences Color::TrueColorRef(idx) via truecolors.get(idx). The reader interns each u32 back into the loading Document's table. Round-trip is lossless and the on-disk size is unchanged. 
u8 tag
   0 = ByLayer
   1 = ByBlock
   2 = Aci(u8) — followed by u8 index
   3 = TrueColor — followed by u32 0x00RRGGBB

§ 9.6 — Lineweight encoding

u8 tag
   0 = ByLayer
   1 = ByBlock
   2 = Default
   3 = Custom — followed by f32 millimetres

§ 9.7 — Geometry encoding

u8 tag
   0 = Line       — 2 × Vec2 (4 × f64 = 32 B)
   1 = Circle     — Vec2 + f64 radius (24 B)
   2 = Arc        — Vec2 + radius + start + sweep (40 B)
   3 = Ellipse    — Vec2 + Vec2 major + f64 ratio (40 B)
   4 = EllipseArc — Ellipse + f64 start_param + f64 sweep_param (56 B)
   5 = Point      — Vec2 location + u8 style + f32 size
   6 = Polyline   — u8 closed + u32 vertex count + (Vec2 + f64 bulge) × n

What's NOT in RSM yet

  • Block table (placeholder in document.rs).
  • Group table.
  • Text styles, dim styles, UCS list, named views.
  • Per-Dobject xdata (DXF application-defined data).

§ 10DXF reader mapping

File: cad_io/src/dxf.rs. DXF is a tagged text format (one group code + value per line). The reader does two passes:

  1. Tokenise: each pair (code, value) becomes Vec<(i32, String)>.
  2. Walk: dispatch on group code 0 (the DXF spec's entity-type marker) and accumulate per-dobject fields until the next 0.

LAYER table mapping

DXF codeField on Layer
2 name
62 color = Color::Aci(abs(value)); visible=false if negative
6 linetype name (looked up in LinetypeTable)
70 flags (bit 0 = frozen)
420TrueColor — NOT yet read on LAYER (follow-up).
430Color book name — NOT supported.

Dobject common-style mapping

Every dobject reader (LINE, CIRCLE, ARC, ELLIPSE, LWPOLYLINE, POINT) shares this Style accumulator pass.

§ 11Default render path

A freshly drawn Line via the Line tool follows this chain end to end.

  1. add_dobject(Geom::Line(...)) →
  2. Document::push(DObject::new(geom)) →
  3. DObject::new calls Style::default():
    • layer = LAYER_BASE, color = ByLayer, linetype = CONTINUOUS, lineweight = ByLayer, visible = true, linetype_scale = 1.0
  4. Document::push rewrites style.layer to layers.active if the active layer differs from LAYER_BASE.
  5. At render time: resolve_color(ByLayer, layer, &layers, &truecolors) → walks to layers[active_layer].color → that's Color::Aci(7) for LAYER B → aci_palette(7) → (255, 255, 255) = white.
Project convention
Dobject by default → drawn on LAYER B (Base) → resolves to color 0 (white) via ByLayer → ACI 7.

§ 12Adding a new per-Dobject property

Seven steps. Follow them in order; missing any step leaves the property silently lost on RSM/DXF round-trip or invisible in the UI.

  1. Struct field — add to Style in cad_kernel/src/style.rs.
  2. Default value — add to Style::default().
  3. RSM I/O — extend write_dobjects and read_dobjects in cad_io/src/rsm.rs. Bump VERSION if the on-disk shape changes.
  4. DXF I/O — extend the style-accumulator in cad_io/src/dxf.rs with the matching group code (e.g. 440 for transparency).
  5. Renderer — the canvas render loop in cad_app/src/app.rs builds (r,g,b) + stroke per Dobject. New properties either modify the resolved style (transparency → alpha) or add a new render channel.
  6. Property panel — keep Dobject_Properties.md in sync.
  7. Tests — kernel test for the resolution chain + RSM round-trip test in cad_io.

§ 13Deferred storage refactors

In rough size-payoff order. None scheduled — trigger only when actual test data shows the ceiling (10 M+ dobjects, or DXF handle preservation needed).

RefactorSaving at 9 M dobjectsTrigger
StyleId indirection — collapse Style to a u32 ref into a shared StyleTable. Two Dobjects with identical style share one row. 252 MB → 36 MB Memory ceiling on real datasets.
SoA Geom storage — Vec<Line>, Vec<Circle> etc. instead of Vec<Geom>. Removes enum discriminant. ~10 % Hot-loop perf profile shows tag dispatch as a bottleneck.
Per-Document handle namespace — DXF round-trip needs handle preservation.
Block / Group tables — first-class containers, not just selection abstractions. N/A (added cost ~64 B / block + refs) Block import / authoring slice opens.
