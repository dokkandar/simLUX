---- (2026-06-25) pline width: DXF/RSM persistence + chamfer continuous + taper fix ----
1. PERSISTENCE (Stage 4): polyline tapered width now survives save/reload.
   - RSM bumped to v4: after the polyline vertices, write u32 count + (start,end)
     f64 per segment; read gated on ver>=4 (older files → empty widths).
   - DXF LWPOLYLINE: write per-vertex 40/41 (start/end width of the segment at

---- (2026-06-24) trim: click a polyline segment with no boundary → remove it ----
When trimming a polyline, clicking a target segment that does NOT intersect any
selected cutter now REMOVES that whole segment (was: kept). Connected-runs path
splits the polyline at the removed segment (before-run + after-run); explode
path drops the clicked segment, keeps the rest. Segment that crosses a cutter
still trims normally. (cad_kernel/src/trim.rs)
App-side (cad_app apply_trim_pick): a POLYLINE target with NO other cutters
(e.g. the polyline itself was the only thing selected) no longer errors "no
other cutters available" — it proceeds with an empty cutter list so the clicked
segment is removed. Non-polyline targets still require a real cutter.
Kernel trim_at top guard ("target has no intersection with the cutting edges")
is now SKIPPED for Polyline targets (the per-segment logic handles miss = remove).
ALL OPEN polylines now go through the connected-runs path (was width-only), so
trimming/removing a segment keeps the rest as a single polyline instead of
exploding to bare Lines (non-width runs carry EMPTY widths). Updated the
trim_polyline test to expect 2 polylines. Closed polylines still explode (v1).

PART 2 — USER-FACING KEYS, COMMANDS & BEHAVIORS (current state)
================================================================================

GLOBAL KEYBOARD KEYS
  Esc        Cancel active tool/command/sub-flow; clear selection; refocus cmd line
  Delete     Erase selected object(s)
  Enter      Finish current drawing (line/polyline/spline); on an EMPTY command
             line, REPEAT the last command; inside a sub-flow, accept default/advance
  Space      Submit the typed command (same as Enter at the prompt)
  Tab        Cycle snap candidates under the cursor
  Ctrl+C / Ctrl+V   Copy / Paste (clipboard events)
  Ctrl+G     Group the current selection
  F7         Toggle grid (env.GrdEnb)
  F8         Toggle CARD — cardinal/ortho drafting lock (cursor H or V only)
  F9         Toggle grid snap (env.GrdSnp)

CONTINUOUS DRAWING (one command, keep going)
  LINE       each click adds a connected vertex; Esc ends.
  POLYLINE (pl)  connected vertices; Enter finishes (open); `c` Enter (or click
                 back on first vertex) closes; Esc cancels.
  SPLINE     connected control points; Enter finishes.
  FILLET     continuous by default — pick pairs until Esc; `r` changes radius
             mid-command (persists as default); `m` single-shot; `t` trim toggle.

SUB-COMMAND KEYS (typed while a command is active)
  PLINE (while drawing):
    A arc mode · L line mode · C close · U undo last vertex · S second-point
    (3-pt) arc · D arc start direction · W/H width / halfwidth
  PEDIT (pe):
    C/O close/open · J join · W width · U undo · X exit
  ZOOM (z):
    A all · C center · E extents · P previous · S scale · W window · O object
    (+ real-time drag)
  FILLET / CHAMFER (while picking):
    R radius (fillet) / D distances (chamfer) · T trim toggle · M multiple toggle
  Selection session (while a command asks you to select):
    p previous selection · l last created object · d remove from selection

COMMAND ALIASES (type then Enter/Space)
  l/line · ci/circle · a/arc · pl/pline/polyline · tr/trim · pe/pedit · z/zoom ·
  group/ungroup · plus tool words (rectangle, spline, …)

PART 3 — KEY ARCHITECTURE NOTES (for re-implementers)
================================================================================
- PLINE WIDTH is a SOLID FILL, not a thick stroke. Data: Polyline.widths
  Vec<(f64,f64)> per segment (start,end), world units, empty = thin. Render:
  cad_app fill_width_strip builds filled convex polygons per segment with
  per-vertex miter (shared apex within limit) or convex-hull bevel (sharp).
- ARC bulge convention: bulge = tan(sweep/4); positive = CCW. bulge_from_arc
  signs by TRAVERSAL direction (CCW-angle vs swept magnitude), NOT chord side
  (chord side is only valid for minor arcs). Arc.sweep_angle stored POSITIVE.
- Arc::reversed() CANNOT encode CW under positive-sweep, so it returns identical
  geometry; chaining reads the far endpoint directly, never via reversed().
- JOIN passes: collinear-line (gap-aware) → concentric-arc → touching-chain →
  polyline. CHAIN_EPS=1e-3 fuzzy endpoint match; JOIN_EPS=1e-6 precision.
- eframe delivers Ctrl+C/X/V as Event::Copy/Cut/Paste, NOT key presses.
